# QuickDesk 信令服务器部署文档

## 快速部署（推荐）

使用 Docker 一体化镜像，包含 PostgreSQL + Redis + 信令服务器，一键部署：

```bash
# 克隆代码
git clone git@gitee.com:Barryda/QuickDesk.git
cd QuickDesk/SignalingServer

# 复制并修改配置
cp .env .env.production
vim .env.production

配置项说明：

**部署必填项（基础设施）：**

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `SERVER_HOST` | 0.0.0.0 | 监听地址 |
| `SERVER_PORT` | 8000 | 服务端口 |
| `DB_HOST` | localhost | 数据库地址 |
| `DB_PORT` | 5432 | 数据库端口 |
| `DB_USER` | quickdesk | 数据库用户名 |
| `DB_PASSWORD` | quickdesk123 | 数据库密码 |
| `DB_NAME` | quickdesk | 数据库名 |
| `REDIS_HOST` | localhost | Redis 地址 |
| `REDIS_PORT` | 6379 | Redis 端口 |
| `REDIS_PASSWORD` | (空) | Redis 密码 |
| `ADMIN_USER` | admin | 初始管理员用户名（仅首次部署有效） |
| `ADMIN_PASSWORD` | admin | 初始管理员密码（仅首次部署有效） |

**可选项（部署后可在管理后台动态配置）：**

以下参数可以在 `.env` 中预设，也可以部署完成后在管理后台「系统设置」页面随时修改，无需重启服务：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `TURN_URLS` | (空) | TURN 服务器地址（逗号分隔） |
| `TURN_AUTH_SECRET` | (空) | TURN 共享密钥（匹配 coturn 的 static-auth-secret） |
| `TURN_CREDENTIAL_TTL` | 86400 | TURN 凭据有效期（秒） |
| `STUN_URLS` | (空) | STUN 服务器地址（逗号分隔） |
| `API_KEY` | (空) | 客户端认证 API Key（空=不启用） |
| `ALLOWED_ORIGINS` | (空) | WebClient 域名白名单（逗号分隔） |
| `ALIYUN_SMS_ACCESS_KEY_ID` | (空) | 阿里云短信 AccessKey ID |
| `ALIYUN_SMS_ACCESS_KEY_SECRET` | (空) | 阿里云短信 AccessKey Secret |
| `ALIYUN_SMS_SIGN_NAME` | (空) | 阿里云短信签名 |
| `ALIYUN_SMS_TEMPLATE_CODE` | (空) | 阿里云短信模板 Code |

> **注意：** `.env` 中的可选参数仅在首次部署时写入数据库。之后所有修改请通过管理后台进行，`.env` 中的值不再生效。
> 
> **短信功能：** 阿里云短信用于手机号验证码登录/注册。四个字段全部填写后短信功能自动启用，任一为空则自动禁用。

---

# 一键部署（使用你的配置文件）
chmod +x deploy.sh
./deploy.sh --env .env.production

# 带域名和 Nginx 反向代理
./deploy.sh --env .env.production --port 8000 --domain your-domain.com
```

或者手动构建和运行：

```bash
# 构建镜像
docker build -t quickdesk-signaling .

# 运行（通过 --env-file 传递配置）
docker run -d \
    --name quickdesk-signaling \
    --restart=always \
    -p 8000:8000 \
    -v /data/quickdesk:/data \
    --env-file .env.production \
    quickdesk-signaling
```

数据持久化在 `/data/quickdesk` 目录，包含 PostgreSQL 和 Redis 数据。

### 部署后配置

部署完成后，请立即登录管理后台完成以下配置：

1. **修改管理员密码**：访问管理后台 → 管理员账户 → 编辑管理员，修改用户名和密码
2. **配置 ICE 服务器**：访问管理后台 → 系统设置 → ICE / TURN / STUN，添加你的 TURN/STUN 服务器
3. **配置安全选项**：按需设置 API Key 和允许的 WebClient 域名
4. **配置短信服务**（可选）：访问管理后台 → 系统设置 → Aliyun SMS，填写阿里云短信的 AccessKey、签名和模板，启用手机号验证码登录功能

这些配置修改后**立即生效**，无需重启服务。

如果需要修改基础设施配置（端口、数据库等），编辑 `.env.production` 后重新部署：

```bash
./deploy.sh --env .env.production
```

## 手动部署（分步安装）

以下是不使用一体化 Docker 的手动安装步骤。

### 系统要求

- CentOS 7/8/Stream 或 Rocky Linux 8+
- 2GB+ 内存
- 10GB+ 磁盘空间
- 公网 IP 和域名（可选）

## 1. 安装 Docker

```bash
# 安装 Docker
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
docker --version
```

## 2. 部署数据库（Docker）

```bash
# 创建数据目录
sudo mkdir -p /data/quickdesk/{postgres,redis}

# 启动 PostgreSQL
docker run -d --name quickdesk-postgres \
  --restart=always \
  -p 5432:5432 \
  -e POSTGRES_USER=quickdesk \
  -e POSTGRES_PASSWORD=quickdesk123 \
  -e POSTGRES_DB=quickdesk \
  -v /data/quickdesk/postgres:/var/lib/postgresql/data \
  postgres:15

# 启动 Redis
docker run -d --name quickdesk-redis \
  --restart=always \
  -p 6379:6379 \
  -v /data/quickdesk/redis:/data \
  redis:7 redis-server --appendonly yes

# 验证运行状态
docker ps
```

## 3. 安装 Go 环境

```bash
# 下载 Go 1.24
cd /tmp
wget https://go.dev/dl/go1.24.0.linux-amd64.tar.gz

# 解压安装
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf go1.24.0.linux-amd64.tar.gz

# 配置环境变量
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
echo 'export GOPROXY=https://goproxy.cn,direct' >> ~/.bashrc
source ~/.bashrc

# 验证安装
go version
```

## 4. 安装 Node.js（构建前端）

```bash
# 安装 Node.js 20 LTS
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs

# 验证安装
node --version
npm --version
```

## 5. 编译信令服务器

> **注意：** 数据库表结构由信令服务器启动时 GORM AutoMigrate 自动创建/更新，无需手动执行 SQL。
> 参考 SQL 见 `SignalingServer/migrations/001_init.sql`。

```bash
# 克隆或上传代码到服务器
cd /opt
git clone git@gitee.com:Barryda/QuickDesk.git
cd QuickDesk/SignalingServer

# 构建前端（管理后台 + 用户门户，Vue 3 + Element Plus）
cd web
npm install
npm run build
cd ..

# 下载 Go 依赖
go mod tidy

# 编译（前端构建产物会通过 go:embed 嵌入二进制）
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
  go build -a -ldflags="-s -w -extldflags '-static'" \
  -o quickdesk_signaling ./cmd/signaling

# 创建运行目录并部署
sudo mkdir -p /opt/quickdesk-signaling
sudo cp quickdesk_signaling /opt/quickdesk-signaling/
```

## 6. 配置并运行服务

```bash
# 复制配置模板到运行目录
sudo cp .env /opt/quickdesk-signaling/.env

# 按需修改配置（数据库地址、密码、TURN 配置、API Key 等）
sudo vim /opt/quickdesk-signaling/.env

# 创建 systemd 服务
sudo cat > /etc/systemd/system/quickdesk-signaling.service << 'EOF'
[Unit]
Description=QuickDesk Signaling Server
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/quickdesk-signaling
ExecStart=/opt/quickdesk-signaling/quickdesk_signaling
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

# 启动服务
sudo systemctl daemon-reload
sudo systemctl start quickdesk-signaling
sudo systemctl enable quickdesk-signaling

# 检查服务状态
sudo systemctl status quickdesk-signaling
```

服务启动后，Go 服务器默认读取工作目录下的 `.env` 文件。也可以通过 `--env` 参数指定任意路径的配置文件：

```bash
# 指定配置文件路径
/opt/quickdesk-signaling/quickdesk_signaling --env /etc/quickdesk/production.env
```

也可以修改 systemd 服务中的 `ExecStart`，添加 `--env` 参数。

后续修改配置只需编辑对应的 env 文件，然后重启服务：

```bash
sudo vim /opt/quickdesk-signaling/.env
sudo systemctl restart quickdesk-signaling
```

## 8. 域名访问配置（Nginx 反向代理）

```bash
# 安装 Nginx
sudo yum install -y nginx
sudo systemctl start nginx
sudo systemctl enable nginx

# 配置反向代理
sudo cat > /etc/nginx/conf.d/quickdesk.conf << 'EOF'
# 关键：根据请求类型动态设置 Connection 头
# WebSocket 请求带 Upgrade 头时设为 upgrade，普通 HTTP 请求设为 close
map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      close;
}

upstream signaling {
    server 127.0.0.1:8000;
}

server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名

    client_max_body_size 100M;

    # WebSocket 端点（长连接）
    # 信令服务器每 60s 发送 WebSocket Ping 帧保活，客户端自动回复 Pong
    # proxy_read_timeout 设为 Ping 间隔的 2 倍以上即可
    location /signal/ {
        proxy_pass http://signaling;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 服务端 Ping 间隔 60s，此处设 300s（5 分钟）留足容错
        # 若 300s 内无任何数据（含 Pong），Nginx 视为连接已死并断开
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;

        # 禁用缓冲，确保实时通信
        proxy_buffering off;
    }

    # HTTP API 和静态文件（普通短连接）
    location / {
        proxy_pass http://signaling;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 30s;
    }
}
EOF

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

## 9. HTTPS 配置（可选）

配置完成后，Nginx 配置会自动更新为：

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    # WebSocket 端点
    location /signal/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
        proxy_buffering off;
    }

    # HTTP API
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

## 故障排查

### 服务无法启动

```bash
# 查看详细日志
sudo journalctl -u quickdesk-signaling -n 100 --no-pager

# 检查端口占用
sudo lsof -i:8000

# 检查数据库连接
docker exec -it quickdesk-postgres psql -U quickdesk -d quickdesk -c "SELECT 1;"
docker exec -it quickdesk-redis redis-cli ping
```

### Nginx 502 错误

```bash
# 检查信令服务是否运行
sudo systemctl status quickdesk-signaling

# 检查 SELinux 状态（可能阻止 Nginx 连接）
sudo setenforce 0  # 临时关闭
sudo sed -i 's/SELINUX=enforcing/SELINUX=disabled/g' /etc/selinux/config  # 永久关闭

# 或者配置 SELinux 允许 Nginx 网络连接
sudo setsebool -P httpd_can_network_connect 1
```

### 数据库连接失败

```bash
# 检查 Docker 容器状态
docker ps -a

# 重启数据库容器
docker restart quickdesk-postgres quickdesk-redis

# 检查数据库日志
docker logs quickdesk-postgres
```

## 安全建议

1. **修改默认密码**：修改 PostgreSQL 和 Redis 密码
2. **配置防火墙**：仅开放必要端口（80、443）
3. **启用 HTTPS**：生产环境必须使用 HTTPS
4. **定期备份**：设置定时任务备份数据库
5. **日志监控**：配置日志收集和告警
6. **限流配置**：在 Nginx 中配置请求限流
7. **客户端认证**：在管理后台「系统设置」中配置 API Key 限制原生客户端访问，配置 Allowed Origins 限制 WebClient 访问

```nginx
# Nginx 限流配置示例
http {
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;
    
    server {
        location /api/ {
            limit_req zone=api_limit burst=20 nodelay;
        }
    }
}
```
