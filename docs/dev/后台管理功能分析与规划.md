### 竞品共同特点

所有竞品（包括开源的RustDesk）都具备的基础管理功能：
- 设备列表的搜索、筛选、排序

- 连接/会话历史日志
- 基本的访问控制

## 四、技术实现要点

### 服务端分页标准接口设计

```
GET /api/v1/admin/devices?page=1&size=20&sort=last_seen&order=desc&search=keyword&os=Windows&online=true

Response:
{
  "devices": [...],
  "total": 536,
  "page": 1,
  "size": 20,
  "totalPages": 27
}
```

### 前端统一列表组件需求

- El-Table + 表头排序（sortable属性）
- El-Input 搜索框（带防抖）
- El-Select 筛选下拉（OS、在线状态、版本、等级等）
- El-Pagination 分页组件
- 导出按钮（调用后端导出接口或前端生成CSV）

## 五、与RustDesk Pro的定位差异

QuickDesk作为自部署远程桌面方案，最直接的竞品是RustDesk Server Pro。RustDesk Pro的管理后台功能相对简单，QuickDesk有机会在以下方面建立优势：

1. **更好的管理体验** — RustDesk Pro的设备管理比较粗糙，QuickDesk可以做到更精细的搜索/筛选/分组
2. **实时监控** — 竞品中只有TeamViewer提供设备级实时监控，这是中小竞品的普遍空白
3. **数据可视化** — 趋势图、连接统计等，让管理员对系统状态一目了然
4. **审计合规** — 操作日志 + 连接历史 + 可选会话录制，满足企业合规需求
